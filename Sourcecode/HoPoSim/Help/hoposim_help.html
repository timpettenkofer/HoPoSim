Tips & Tricks

Use tooltips to get contextual help

Most input user interface elements provide a tooltip that appears when hovering the mouse over it.
These tooltips provide you with a hint about the item being hovered over.

Use contextual undo in input elements

Most input user interface elements provide an undo functionality (via the Ctrl-Z shortcut) to erase the last change done in this element, reverting its value to an older state.

Software updates

To get the latest update available for your software, just navigate to the Settings (Einstellungen) and browse down to the Software-Upgrade section. Click then on the 'Jetzt auf Updates prüfen' button. The software will automatically check whether your version is up-to-date or if an update is available and can be installed.

Use navigation bar to navigate between views

You can use the backward and forward arrows on the top left corner of the application's window to go back/go forward one page in your view's history.

Shortcuts:
"Alt + ←" or "Backspace" to navigate back

"Alt + →" or "Shift + Backspace" to navigate forward

Use shortcuts to navigate between tabs

You can use Ctrl + Tab navigate between tab items and Enter to select them.

FAQs

How to get an empty template for manual collection of wood logs data?

Go to the Home view and click on the Eingabevorlage tile. This will generate an empty template in the desired destination folder.
Once filled up with data this file can be directly imported in the software via the Eingabe view.

How to automatically generate a data set?

To generate a new excel data set, first create a new item in the Generator View and select the number of classes you need for the different available modelling parameters (diameter, tapering, bending and ovality).
The generator UI allows you then to define the number (and length) of the wood logs you want to be generated in the data set.

A table is then displayed for each modelling parameter. It allows you to define the min/max values (von/bis) for each class as well as their repartitions values (Anteil % or N) for each node of the tree hierarchy.

Class headers: The class headers are buttons that allow you select a given node and help you navigate in this tree hierarchy (diameters at the top-level, tapering at the second, bending nodes at third level, etc...).

Header colors help you along the way by displaying an orange (uninitialized), green (valid sub-tree values) or red (invalid sub tree values) color for each class.
Note that the amount of wood logs for a given class at a given level should match the sum of wood logs' amounts on the level below.
For example, given an amount (Anteil N) of 10 for Mittendurchmesserklasse 1, the sum of wood logs distributed at the Abholzigkeitslevel below should match 10.

Once you defined the min/max (von/bis) and repartitions values (Anteil % or N) for each classes and all headers are green you can generate a wood log list in excel format with the help of the command on the left side of the view (Einzelstammdaten generieren).

Note that Wurzelanlauf/Stammfuß allows you to generate values for root flares modeling.
Here you can specify a proportion (Anteil %), i.e. the percentage of wood logs that should have root flares, and a height interval (Höhen in cm) determining the interval in which random values for the root height will be generated in the Stammliste file for each wood log that should be quipped with root flares.

How to load a data set?

To load a given HoPoSim data set either automatically or manually generated, first create a new item in the Eingabe View and trigger the Stammdaten importieren button in the left panel of the view.
Once loading is complete, the imported is displayed in the Stammliste section of the view.

How to prepare a data set for simulation?

Once your data set is imported as described above, you can configure the simulation parameters (Polterungsparameter) by providing values for the parameters appearing in the "Polterung" and "Holz" section (use the tooltips on the input fields to get a hint for each parameter).
Parameters in the Polterung section allow you to configure the characteristics of the Polter you want to simulate (Size, Polterunterlage, Quality, ...).
In the Holz section you can globally specify the characteristics of the wood itself (Tree species, Wood and Bark characteristics, wood logs' bending, etc...).
Once configured select the Simulation konfigurieren command in the left panel. This will generate a simulation configuration that can be used in the Simulation section.

How to start a simulation?

Given a simulation configuration generated as described above, select a number of iterations (Iterationen) and quality levels (Genauigkeit, Fotooptische Genauigkeit) and select the Simulation starten command in the left panel.

The Genauigkeit slider controls how many Stirnpunkte points are take into account for the computation of the Fotooptik, Sektionraummaß and Polygonzug values. This ranges from a quarter (Hoch) to a seventh part (Niedrig) of all the points. A high quality means more accurate results but longer computing times.

The Fotooptische Genauigkeit slider controls the expected average number of points (Stützpunkte) per running meter (Laufmeter) for the construction of the Fotooptic hull. A value of zero results in a convex envelope. Higher values generate more accurate hulls (i.e. with smaller areas), as concave angles are allowed.

What are the computed iteration values?

For each iteration the following values are computed and displayed in the view table:

  • #: Iteration ID
  • Status: Iteration Ergebnis (OK, Timeout, Error)
  • Stirnfläche V (qm): Polterfläche Vorderseite in Quadratmeter m. R. (Summe der Stammstirnflächen)
  • Stirnfläche R (qm): Polterfläche Rückseite in Quadratmeter m. R. (Summe der Stammstirnflächen)
  • Fotooptik V (qm): Polterfläche Vorderseite in Quadratmeter (errechnet durch Polygonzug auf der Poltersvorderseite mit eingestellter Fotooptische Genauigkeit)

    = Fläche(Hülle_Vorderseite)

  • Fotooptik R (qm): Polterfläche Rückseite in Quadratmeter (errechnet durch Polygonzug auf der Polterrückseite mit eingestellter Fotooptische Genauigkeit)

    = Fläche(Hülle_Rückseite)

  • Fotooptik (Rm): Poltervolumen in Raummeter m. R. (errechnet durch Polygonzug mit eingestellter Fotooptische Genauigkeit)

    = Poltertiefe * (Fotooptik_V + Fotooptik_R) / 2

  • Stützpunkte V: Anzahl der Stützpunkte des Polygonzuges um die Poltervoderseite
  • Stützpunkte R: Anzahl der Stützpunkte des Polygonzuges um die Polterrückseite
  • Polygonzug V (qm): Polterfläche Vorderseite in Quadratmeter m. R. (errechnet mittels Polygonzug mit eingestellter Genauigkeit)
  • Polygonzug R (qm): Polterfläche Rückseite in Quadratmeter m. R. (errechnet mittels Polygonzug mit eingestellter Genauigkeit)
  • Polygonzug (Rm): Poltervolumen in Raummeter m. R. (errechnet mittels Polygonzug)

    = Poltertiefe * (Polygonzug_V + Polygonzug_R) / 2

  • Sektionsraummaß V (qm): Polterfläche Vorderseite in Quadratmeter m. R. (errechnet mittels Sektionsraummaß nach Vorgaben der RVR, ohne Berücksichtigung von Reduktionsfaktoren)
  • Sektionsraummaß R (qm): Polterfläche Rückseite in Quadratmeter m. R. (errechnet mittels Sektionsraummaß nach Vorgaben der RVR, ohne Berücksichtigung von Reduktionsfaktoren)
  • Sektionsraummaß (Rm): Poltervolumen in Raummeter m. R. (errechnet mittels Sektionsraummaß nach Vorgaben der RVR, ohne Berücksichtigung von Reduktionsfaktoren)

    = Poltertiefe * (Sektionsraummaß_V + Sektionsraummaß_R) / 2

  • Poltervolumen o.R (Fm): Poltervolumen in Fm ohne Rinde, errechnet aus der Summe aller Einzelstammvolumen mit Rindenanteil
  • Poltervolumen m.R (Fm): Poltervolumen in Fm mit Rinde errechnet aus der Summe aller Einzelstammvolumen ohne Rindenanteil
  • Polterunterlage o.R (Fm): Polterunterlagenvolumen in Fm ohne Rinde errechnet aus der Summe der Stammvolumen der Polterunterlage mit Rindenanteil
  • Polterunterlage m.R (Fm): Polterunterlagenvolumen in Fm mit Rinde errechnet aus der Summe der Stammvolumen der Polterunterlage ohne Rindenanteil
  • Rindenanteil (%): Rindenanteil in % des Festgehalts/Raumvolumens mit Rinde
  • UF Sektion. o.R: Umrechnungsfaktor errechnet aus dem Poltervolumen in Fm o. R und dem Poltervolumen (errechnet mittels Sektionsraummaß) in Rm m. R.

    = Poltervolumen o.R / Sektionsraummaß_Rm

  • UF Sektion. m.R: Umrechnungsfaktor errechnet aus dem Poltervolumen in Fm m. R und dem Poltervolumen (errechnet mittels Sektionsraummaß) in Rm m. R.

    = Poltervolumen m.R / Sektionsraummaß_Rm

  • UF Polygonzug o.R: Umrechnungsfaktor errechnet aus dem Poltervolumen in Fm o. R und dem Poltervolumen (errechnet mittels Polygonzug) in Rm m. R.

    = Poltervolumen o.R / Polygonzug_Rm

  • UF Polygonzug m.R: Umrechnungsfaktor errechnet aus dem Poltervolumen in Fm m. R und dem Poltervolumen (errechnet mittels Polygonzug) in Rm m. R.

    = Poltervolumen m.R / Polygonzug_Rm

  • UF Fotooptik o.R: Umrechnungsfaktor errechnet aus dem Poltervolumen in Fm o. R und dem Poltervolumen (errechnet mittels Fotooptik) in Rm m. R.

    = Poltervolumen o.R / Polygonzug_Rm

  • UF Fotooptik m.R: Umrechnungsfaktor errechnet aus dem Poltervolumen in Fm m. R und dem Poltervolumen (errechnet mittels Fotooptik) in Rm m. R.

    = Poltervolumen m.R / Polygonzug_Rm

  • Höhe: Geschätzte Höhe des Polters in Meter errechnet als Durchschnitt der 3 mittleren Sektionen, die über den Sektionsraummaß erhalten wurden.
  • Breite: Breite der Bounding-Box des gesamten Polters (ohne Polterunterlage) in Meter.

How to visualize/export a given simulation iteration?

Right click on the iteration you want to inspect in the view results' table', this will open a context menu containing the following commands:

  • Visualisieren: opens a 3D interactive visualization of the iteration.
  • Exportieren/Als Bild: exports the iteration as a front and a back image in PNG format.
  • Exportieren/Als 3D-Modell (obj): exports the iteration as a 3D model in Wavefront OBJ format.
  • Exportieren/Als 3D-Modell (fbx): exports the iteration as a 3D model in Autodesk FBX format.

Note that you can configure the size (width and height) of the generated images in the Einstellungen/Export section.

How to specify/customize the characteristics of a tree species (Baumart)?

Go the 'Einstellungen' page of the application, then select 'Legenden' on the left side of the page.
Under the section 'Baumarten' you can assign your fine tuned values to the parameters exposed in the table:

  • Name: The name of the tree species (read only)
  • Wurzelanlauf: Specifies whether this tree species supports root flares
  • Min Wurzelknoten: The minimum number of root flares this tree species can have
  • Max Wurzelknoten: The minimum number of root flares this tree species can have
  • Min Wurzelknoten Radius: The minimum radius multiplier (with respect to the wood log diameter) for root flares. For example if set to 2, the root flares should at least be two times wider than the wood log itself.
  • Max Wurzelknoten Radius: The minimum radius multiplier (with respect to the wood log diameter) for root flares
  • Aststummel: Specifies whether this tree species supports branch stubs
  • Anteil Stämme mit Aststummel: Specify the number of wood log (in %) for a given Polter that should be modeled with branch stubs. For example given a value of 70% and Polter of 100 wood logs, 70 wood logs will be equipped with branch stubs.
  • Aststummel Min Länge: The minimum length of a branch stub (measured perpendicular from the wood log itself)
  • Aststummel Max Länge: The maximum length of a branch stub (measured perpendicular from the wood log itself)
  • Anzahl Aststummel per Meter: The number of branch per lfm for this tree species. For example a value of 0.6 will give 2 stubs for a 3m long wood log.
  • Aststummel Radius: The relative radius multiplier (with respect to the wood log diameter) for branches.
  • Aststummel Min Höhe: The minimum height (with respect to the wood log's length, i.e.factor between 0 and 1) where branch stubs appear on the wood log. A value of 0.2 for a 5m long wood log will only allow branches after the first meter.
  • Aststummel Max Höhe: The maximum height (with respect to the wood log's length, i.e.factor between 0 and 1) where branch stubs appear on the wood log.
  • Aststummel Winkel: The average angle (factor between 0 and 1) made by the branches with respect to the wood log. A value of 1 will generate horizontal branches, a value of 0 will generate branches pointing up.
